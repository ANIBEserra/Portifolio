Extraction
01 — Data Sourcing

Large Scale API Consumption

Development of a data collection engine in Python to interact with BrasilAPI, extracting data from 55 CNPJs.

  • Request and retry handling
  • Collection of demographic and registration data
  • Initial structuring in JSON/Dictionary format
  • Dependency isolation via Docker
Organization
02 — Data Architecture

Partitioning Strategy

Implementation of temporal partitioning logic to ensure query efficiency and organization of the Bucket.

  • Hierarchical organization by Year/Month/Day
  • Standardization for Hive-style partitioning
  • Ease of search for temporal snapshots
  • Optimization of storage and scan costs
Cloud Loading
03 — Cloud Storage

Hydration of the Data Lake (GCS)

Secure connection with Google Cloud Platform for automatic loading of processed data.

  • Integration via Google Cloud SDK
  • Management of Service Accounts and Secrets
  • Optimized upload for GCS Buckets
  • Maintenance of file integrity in the Load
Automation
04 — Orchestration

GitHub Actions (Lite Version)

Configuration of a CI/CD workflow to transform the script into a serverless and fully autonomous process.

  • Scheduling via Cron (daily at 08:50 AM BRT or UTC 11:50 AM)
  • Instantiation of temporary runners
  • Configuration of secure environment variables
  • "Hands-off" pipeline (no manual intervention)
Next Steps
05 — Scalability

Airflow & Cloud Expansion

The project is ready to evolve from a Lite model to a robust orchestration using complex containers.

  • Advanced orchestration via Apache Airflow
  • Deploy on Google Compute Engine (VM)
  • Real-time container health monitoring
